# Agent Delegation

Use specialists for execution. Keep Garion focused on orchestration, decisions, and quality control.

## Rules
- one writer per task artifact
- define objective, context, success criteria, and output location
- require proof before completion
- use Beldin-style review for high-risk work

## Typical Routing
- implementation -> Silk
- research -> Barak
- content -> Polgara
- UX/strategy -> Ce'Nedra
- growth -> Relg
- analytics -> Taiba
- QA/risk review -> Beldin

## Artifacts
- tasks -> `shared/tasks/`
- outputs -> `shared/outputs/`
- operational status -> `ops/status/`
